Usage notes

  • Designed to scroll horizontally if all columns cannot fit within viewport.
  • Responsive tables are highly preferred over this pattern. Use this for situations where a tabular format is essential even on mobile or there is no way to limit the number of columns.
  • First column and column headers are fixed when scrolling. Widths will need to be adjusted in the CSS.
  • Scroll indicators and shadows reinforce that additional content is available.
  • Keyboard accessible: Table container is focusable; arrow keys arrow scrolling.
  • Touch scrolling optimized for mobile.
  • Required caption available to screen readers but accessibly hidden from visual display.
  • Columns have set widths and alignment; adjust to accommodate your needs.
  • Link/focus behaviour will inherit from main stylesheet; make sure those :link, :hover, and :focus states are accessible.
  • Table will expand to full available width; add constraining container or max-width if needed.
