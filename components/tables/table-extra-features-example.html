Usage notes

  • Shows preferred way to allow sorting by column and pagination format.
  • Screen reader announcements for all state changes via aria-live regions.
  • 44px minimum touch target size for mobile accessibility.
  • Focus management moves to table after page navigation.
  • Pagination shows set number of records per page with first/previous/next/last navigation and ellipses for large result sets. (Tables would normally have more than 5 records per page; this limit is for demonstration purposes.)
  • Table rows stack on small screens, up to 768px.
  • Useful for basic text/data tables with no actions or a link to a detail page. (If the interaction is to a popup or modal, use a button instead of a link; there is a pattern available for that.)
  • Includes optional status pills for conditions or statuses:
    • Positive (green) = completed, on-track; positive state
    • Negative (red) = warning, error, off-track; negative state
    • Pending (blue) = pending; neutral condition
    • In-progress (yellow) = in progress; neutral or alert condition
  • Required caption available to screen readers but accessibly hidden from visual display.
  • Link/focus behaviour will inherit from main stylesheet; make sure those :link, :hover, and :focus states are accessible.
  • Table will expand to full available width; add constraining container or max-width if needed.
