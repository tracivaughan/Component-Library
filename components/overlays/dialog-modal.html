About Dialog Elements

The HTML dialog element represents a modal or non-modal dialog box or other interactive component.

Key benefits include:

  • Built-in accessibility features
  • Automatic focus management
  • ESC key handling
  • Backdrop styling with ::backdrop
  • Form integration capabilities

Usage notes

  • This modal uses the native dialog, along with javascript to handle the focus trapping.
  • Instructions:
    1. Put the ptn-opendialog-btn class on the trigger button.
    2. Add the dialog markup anywhere in the HTML. Each modal must have a unique ID that must be added to the trigger's custom data attribute (data-dalog).
    3. Add an ID to the dialog header and reference it in the dialog's aria-labelledby.
  • You can adjust the formatting in the CSS, including adding additional classes to visually distinguish different modals (for example, errors, warnings, confirmations could be themed).
