Native HTML Accordion - No JavaScript Required

What is this?

This accordion uses the native HTML <details> and <summary> elements. It requires no JavaScript and works even if JavaScript is disabled or fails to load.

Quick Start - Copy & Paste This:

<div class="ptn-native-accordion">
    <details>
        <summary>Your Question Here</summary>
        <div>
            <p>Your answer here</p>
        </div>
    </details>
</div>

No JavaScript needed. Works immediately.

Example 1: Standard FAQ Style

What are details and summary elements?

<details> is a native HTML element that creates a disclosure widget where information is visible only when toggled. The <summary> element specifies the visible heading.

These elements have been part of the HTML standard since HTML5 and are supported in all modern browsers.

What are the advantages of native elements?
  • No JavaScript required - Works even if JS fails or is disabled
  • Built-in accessibility - Screen readers understand it natively
  • Semantic HTML - Clear meaning to browsers and assistive technology
  • Progressive enhancement - Core functionality always works
  • Smaller file size - No additional JavaScript to download
  • Better performance - Browser-native implementation is optimized
Can I use keyboard navigation?

Some keyboard support is built-in:

  • Tab - Navigate to the summary element
  • Enter or Space - Toggle the details open/closed

Arrow keys, Home, and End keys are not supported, because each disclosure element is seen as an indvidual. Use the ARAI-enhanced accordion if you want full keyboard support.

How do I make one open by default?

Just add the open attribute to the <details> element:

<details open>
    <summary>This will be open by default</summary>
    <div>Content here</div>
</details>

(Notice this panel was open when you loaded the page!)

Comparison: Native vs JavaScript Accordion

✅ Native Advantages

  • No dependencies, zero JavaScript
  • Smaller file size
  • Better performance
  • Built-in accessibility
  • Simpler code
  • Progressive enhancement

❌ Native Limitations

  • No exclusive mode (without JS)
  • No bulk expand/collapse (without JS)
  • More limited keyboard navigation
  • Limited animation control
  • Browser styling differences
  • Marker styling quirks
  • No custom events

Usage Guide

Basic HTML Structure

<details>
    <summary>Visible heading</summary>
    Hidden content goes here
</details>

With Wrapper Div (Recommended)

<details>
    <summary>Question</summary>
    <div>
        <p>Answer with proper padding</p>
    </div>
</details>

Using a wrapper <div> makes padding and styling more predictable.

Open by Default

<details open>
    <summary>This starts open</summary>
    <div>Content</div>
</details>

Styling Classes

  • .ptn-native-accordion - Standard bordered container style with arrows
  • .ptn-native-simple - Simple style with plus/minus icons

Key CSS Techniques

  • Remove default marker: Use list-style: none and target ::-webkit-details-marker and ::marker
  • Add custom icon: Use summary::after with positioned content
  • Animate icon: Apply CSS transitions to summary::after
  • Style open state: Use details[open] selector

Accessibility Features (Built-in)

  • Keyboard accessible (Tab, Enter, Space)
  • Screen reader announces expanded/collapsed state
  • Semantic HTML - assistive tech understands the structure
  • Focus management handled by browser
  • No ARIA attributes needed

Common Gotchas

  • Marker removal: You need both ::marker and ::-webkit-details-marker to hide the default triangle in all browsers
  • Content wrapping: Use a wrapper div inside details for better padding control
  • Smooth animations: Native details doesn't support smooth open/close animations easily (requires JavaScript or newer CSS features)
  • Exclusive mode: Multiple details can be open simultaneously - need JS to make them exclusive

Combining with JavaScript (Optional Enhancement)

You can enhance native details with JavaScript for additional features while maintaining the no-JS fallback:

// Optional: Add exclusive mode with JS
document.querySelectorAll('details').forEach(detail => {
    detail.addEventListener('toggle', () => {
        if (detail.open) {
            // Close other details in the same container
            const container = detail.parentElement;
            container.querySelectorAll('details[open]').forEach(other => {
                if (other !== detail) other.open = false;
            });
        }
    });
});
