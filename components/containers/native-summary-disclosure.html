Simple Disclosure component: Native disclosure elements

A disclosure widget is a control whose sole purpose is to hide and disclose (or show) content. It is implemented as a button (trigger) that, when activated, toggles the display of some content (content panel).

According to the HTML Standard:

The details element represents the disclosure widget container, and the first summary element child will be the disclosure widget's trigger.

Usage notes

  • This can be used for the simplest situations where you need to show/hide information. For example: hiding the full content of an article, or toggling details.
  • Screen readers do vary slightly in how they read the trigger. It's not a bug, and isn't something to wrorry about.
  • Like other native HTML controls, the browser bakes keyboard interactions into the summary element by default. This means that, like a button, it is operable using both Space and Enter keys.
  • Be sure to provide only ONE Summary element in the component.
  • When you search for content on a page using the browser's “Find in page” search functionality, Chromium browsers (like Chrome and Edge) will automatically expand the details panel if it contains your search terms. If this behavior is not desired, use the custom aria version.
  • If you customize the trigger image, use an inline SVG. Other methods can result in odd readings of the trigger. Inline SVG is also more flexible in terms of styling, animating, as well as adapting the icon for Forced Colors Modes like Windows High Contrast Mode.
  • By default, the browser will treat the plain summary as a block element (focus on the first example above). You can change the display value if desired.
